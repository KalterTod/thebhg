Experimental Roster 3 XML Interface

This is an experimental read-only interface to the BHG Roster 3 via XML. Available files and their options follow:

  • person.php (Example)
    Returns an XML representation of a Person object, along with their rank, position, and division information. Requires one parameter: id (the roster ID of the person).

  • division.php (Example)
    Returns an XML representation of a Division object, including the roster IDs of its members, which could then be looked up using person.php. Requires one parameter: id (the roster ID of the division).

  • cadre.php (Example)
    Returns an XML representation of a Cadre object, including the roster IDs of its members, which could then be looked up using person.php. Requires one parameter: id (the roster ID of the cadre).

  • rank.php (Example)
    Returns an XML representation of a Rank object. Requires one parameter: id (the roster ID of the rank).

  • position.php (Example)
    Returns an XML representation of a Position object. Requires one parameter: id (the roster ID of the position).

  • search.php (Example)
    Provides a hook into the Search* methods of a Roster object. Requires two parameters: type (the type of search to conduct; one of "name", "ircnick", "email", "position", "rank"); value (the value to search for).

Additionally, all files accept a text parameter, which if set will cause the script to set the Content-Type to text/plain instead of text/xml.

At this stage, I'm afraid you'll have to look at the XML files themselves to figure out the tags and so on, but it's all pretty simple, particularly if you're familiar with the Roster 3 PHP interface. There are no plans for write access, nor are there plans for further files at this stage. Note, however, that this is experimental, and subject to change at any point. Also, it's not part of the official Roster, and if it breaks, it's a very low priority.

So, with all those disclaimers out of the way, have fun. You can view some sample code here.
